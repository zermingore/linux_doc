Introduction

Le rôle d'un compilateur, c'est de prendre du code et de génerer un binaire que ton CPU peut comprendre. Un compilateur d'un language de programmation relativement moderne est un programme conmplexe (genre vraiment complexe, je classe ça top 3: Kernel, Navigateur Web, Compilateur).
Bonne nouvelle: Pour comprendre en gros comment ça marche, pas besoin de sauter par la fenêtre.
Mauvaise nouvelle: Si tu veux comprendre à 100% comment ça marche, là par contre, c'est possible mais vraiment hard-core. J'aimerai bien prender le temps de piger mais ça c'est largement aussi au-dessus de mes moyens pour l'instant.

Front End

Pré-Processeur

Le compilateur va faire quasi litéralement du search-and-replace sur ton code. "#include <stdio.h>" -> ok j'ai trouvé /usr/include/stdio.h je copie-colle son contenu là où t'as ta directive "#include"
Si t'as des macros genre genre "#ifdef __WINDOWS__ .... #endif" le pré-processeur est en mode "ok, tu m'as pas demandé de compiler pour Windows du coup ça ça dégage"
Si t'as des macros genre genre "#ma_constante_moche 42" le pré-processeur remplace toutes les occurences de ma_constante_moche par 42, sans réflechir, c'est pas son boulot
Fun fact: tu peux demander au compilateur de s'arrêter après la phase de pré-processing.
$> gcc -E test.c
