man

Peut-être la command la plus utile sous Linux.
Oui ça fait vieux con de dire "lis la documentation (Read The Fucking Manual -> RTFM en anglais)". Mais c'est généralement LA source la plus actuelle et celle le moins sucéptible de dire de la merde.
De plus, ce manuel est installé par défaut sur ton système et presque chaque programme installé vient avec ses pages de manuel.
Ces page de manuels sont souvent écrites et maintenues par les dévelopeurs des programmes en question et, souvent modifiées par des gens qui trouvent que quand quelque-chose n'est pas clair modifient ou demandent aux developeurs de modifier ces pages.

En règle plus générale, quelle que soit la commande; d'où qu'elle proviennent; c'est vraiment une TRÈS bonne idée de chercher dans la doc de la commande en question pour apprendre un truc au passage. Si j'écris "et là on fait un ls -1t", cool c'est ls on peut rien casser. Oui mais d'après le man ls "-1 list one file per line." et "-t sort by time, newest first". Voila qui peut être pratique.
Quand, par exemple, dans un long script tu tombe sur du, les premières lignes de la description suffisent très souvent. Pour du "Summarize disk usage...". Ok, ça parle de place sur le disque; Pas forcément besoin tout de suite de checker toutes les options possibles ou utilisés dans le script en question.

grep

Packages manager (gestionaire de programmes)

Un gestionnaire de paquets utilise une base de données de programme installable. Tu peux voir ça exactement comme un store sur téléphone.
Tu peux installer un tas de programme sans devoir te battre à essayer de trouver le site officiel, vérifier que le binaire est compatible avec ton architécture, etc.. Ces problèmes sont gérés par le gestionaire de paquets.
Sous les systèmes basés sur Debian (le cas de ce qui tourne sur ta Pi), ce gestionaire s'appelle apt (pour Advanced Package Tool).
Concrètement, si tu cherches un programme, apt search mot_cle (par exemple apt search raspberry si tu cherche un truc qui a un lien avec une Raspberry)
Si tu veux installer quelque-chose: apt install nom_programme (par exemple apt install git)
Note: ton gestionaire de paquets fait mumuse avec ton système (par exemple, copie des fichiers dans /usr/bin/). C'est pourquoi installer quelque nécessite d'être root (administrateur, utiliser sudo).
